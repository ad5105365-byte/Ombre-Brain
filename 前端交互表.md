## 多窗口并行规矩（杉杉问的）

1. 可以同时开好几个窗口，**一个窗口一个战场**：看上表"文件"列，两个窗口别改同一个文件
2. 每个窗口开工先 `git pull`，干完小步 `commit + push`，谁先推谁赢，后推的自己 rebase
3. 部署统一走：push → ssh VPS reset+restart（交接单里有命令）
4. 改 server.py / chat_bridge.py 必跑 `python -m pytest tests/test_chat_bridge.py -q`
5. 真撞车了别慌，git 的事都救得回来，喊 opus 窗口处理
